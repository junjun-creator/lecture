자바는 태생적으로 모든 가전에 소프트웨어를 넣기위해(WORA를 지원하기 위해) 태어났다.
운영체제가 다른 것에 대한 문제점을 api를 통해 해결하였고
	> 운영체제마다 다른 api를 맞출수 있는 어댑터 역할하는 자바 플랫폼 개발
CPU가 다른것에 대한 문제점을 중간코드(바이트코드로 뿌려줌)로 해결
	> cpu가 다른문제를 해결하려면 소스코드를 그대로 보내주고 알아서 니들이 맞게 써라 라고
	해야하는데, 그러면 소스코드가 공개되는문제와, 수행속도가 느려진다는 문제가 있다
	이를 해결하기 위해, 중간코드로 컴파일러를 통해 번역하고, 그 코드를 보냄으로써
	단점을 극복 한다.
		>>중간코드를 갖고 있는게 class 파일이다.

우리가 만든 Program.class파일은 현실에서 보기엔 데이터 파일인데,
java.exe(자바세상에서)가 바라보는 Program파일은 실행파일인 것이다.
java.exe는 실행 파일을 원한다. 즉 Program파일은 자바 실행 파일이다.
디렉토리 > java ..\..\..\workspace\Program 이렇게 되면 에러가 발생하는데,
java.exe는 뒤에 입력되는 문자열(실행파일명)은 경로가 포함되든 말든  그건 무조건 실행파일 명으로만 인식
그럼 얘는 이경우에 Program을 어디서 찾는가?
>> 현재 디렉토리, 또는 클래스 path 환경(운영체제 path환경 아님)에서 찾는다.
클래스 패스를 지정하는 방법은
>java -cp "경로" Program 으로 작성해주어서 클래스 path를 지정해주어야 한다.

또 다른 관점에서, 컴파일러(javac)없이, 오로지 소스코드만을 가지고 바로 실행(java, 인터프리터)도 가능하다

입,출력은 자바 언어에 예약어로 존재 하지 않는다. 라이브러리를 사용한다.
입출력 장치는 종류가 너무 다양하고, 호환되기 쉽지 않다.(변화가 다양하기때문)
입출력 장치를 언어가 직접 사용하게끔 하면 안된다. 직접 사용하게 되면 변화가 생겼을 때 문제가 생긴다
기능단위로 바라보고 간접적으로 그 기능을 사용하게끔 하게 하면 된다.
간접적으로 사용할수 있는 것을 인터페이스라고 한다.

인터페이스는 함수로 구현되는데 함수는 식을 깔끔(중복제거)하게 만들기 위해서 사용된다.
C언어는 운영체제의 api를 직접 사용하는데, 이렇게 되면 운영체제별로 다른 api를 사용해야한다.
java는 WORA를 구현하기 때문에, JAVA 플랫폼 api를 사용하여 모든 운영체제와 호환한다.

API란? application programming interface. 앱을 만들때 사용하는 인터페이스
그럼 interface(대리자)는 왜 사용하는가? 직접 대상을 사용할수 없는 상황(변화가 잦는 등)을 피하기 위해
그렇다면 Social API는 무엇인가? 
카카오 api, 페이스북 api 등등 에도 입출력 기능을 사용하고 싶은데, 이 것을 사용하려면 
카카오, 페이스북을 노출 하게 해야하는데, 노출은 싫으니.... 중간다리인 api를 만들어서 제공한다!
(보안공간에 들어갈때 가방을 맡긴다고 하면, 직접 사물함에 넣을건지 , 관리자에게 맡기는지 비교하면 됨)

대상 내부의 내용을 알 필요도 없고, 대상의 변화에도 아무런 영향을 받지 않는다

자바 언어와 플랫폼을 가지고 절차를 만든다
자바 언어는 메모리와 cpu를 다루는 도구이고,
플랫폼은 입력 출력과 같은 외부 장치를 사용하는 도구이다.
플랫폼 api를 사용함으로써 사용하는 대상을 알 필요도 없고 그 대상의 변화에도 영향을 받지 않는다.
